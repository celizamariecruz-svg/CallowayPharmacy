üì¶ What is a Variable?

A variable is like a labeled container that holds information. You can put data in, take data out, or change what's inside.

üì¶

$medicine_name

Container Label

"Biogesic"

What's stored inside

üí° Real World Analogy

Think of variables like labeled jars in a pharmacy shelf:

  • Jar Label = Variable Name ($medicine_name)
  • Contents = The Value ("Biogesic")
  • You can replace the contents, but the label stays the same

üè∑Ô∏è Data Types (What Kind of Data)

Variables can hold different types of data. PHP automatically figures out the type (it's "loosely typed").

üìù String - Text/Words

Any text wrapped in quotes. Can be letters, numbers, symbols - anything!

$name = "Juan Dela Cruz"; // Double quotes $address = 'Manila, Philippines'; // Single quotes $phone = "0917-123-4567"; // Numbers as text $mixed = "Order #12345"; // Mixed text

Used for: Names, addresses, descriptions, any text

üî¢ Integer - Whole Numbers

Numbers without decimal points. Can be positive or negative.

$quantity = 50; // Positive number $stock = -10; // Negative number $product_id = 12345; // ID number $year = 2025; // Year

Used for: IDs, quantities, counts, ages

üí∞ Float/Double - Decimal Numbers

Numbers with decimal points. Used for money, measurements, percentages.

$price = 99.99; // Price in pesos $discount = 0.15; // 15% as decimal $temperature = 36.5; // Body temp $average = 85.75; // Average score

Used for: Prices, percentages, measurements, calculations

‚úÖ Boolean - True or False

Only two possible values: true or false. Used for yes/no questions.

$isLoggedIn = true; // User is logged in $isExpired = false; // Product not expired $hasDiscount = true; // Discount applies $inStock = false; // Out of stock

Used for: Status flags, toggles, conditions

üìã Array - List of Items

A collection of multiple values in one variable. Like a shopping list!

// Simple list (indexed array) $medicines = ["Biogesic", "Neozep", "Bioflu"]; // List with labels (associative array) $product = [ 'name' => "Biogesic", 'price' => 5.50, 'stock' => 100 ];

Used for: Lists, collections, database records

‚¨ú NULL - No Value

Means "nothing" or "empty". Not zero, not empty string - actually nothing.

$middleName = null; // No middle name $discount = null; // No discount applied $result = null; // Not computed yet

Used for: Optional fields, unset values, "not applicable"

üìã Arrays Explained (Super Important!)

Arrays are used EVERYWHERE in your pharmacy system. Let's understand them well!

Indexed Array (Simple List)

Items are numbered starting from 0.

// Creating a list of medicines $medicines = ["Biogesic", "Neozep", "Bioflu", "Solmux"]; // ‚Üë ‚Üë ‚Üë ‚Üë // Index 0 Index 1 Index 2 Index 3 // Access items by index number echo $medicines[0]; // Output: Biogesic echo $medicines[2]; // Output: Bioflu // Get how many items echo count($medicines); // Output: 4
‚ö†Ô∏è Remember: Counting starts at 0!

First item is [0], second is [1], third is [2], etc. This confuses beginners a lot!

Associative Array (Labeled List)

Items have text labels (keys) instead of numbers.

// Creating a product with labeled data $product = [ 'name' => "Paracetamol 500mg", 'brand' => "Biogesic", 'price' => 5.50, 'stock' => 150, 'expiry_date' => "2026-12-31" ]; // Access items by key name echo $product['name']; // Output: Paracetamol 500mg echo $product['price']; // Output: 5.50 // Update a value $product['stock'] = 145; // Now stock is 145

Array of Arrays (2D Array)

When you get data from database, you usually get an array of arrays - a list of products.

// List of products (like from database) $products = [ ['id' => 1, 'name' => "Biogesic", 'price' => 5.50], ['id' => 2, 'name' => "Neozep", 'price' => 8.00], ['id' => 3, 'name' => "Bioflu", 'price' => 12.00] ]; // Access: First the row [0], then the column ['name'] echo $products[0]['name']; // Output: Biogesic echo $products[1]['price']; // Output: 8.00 // Loop through all products foreach ($products as $product) { echo $product['name'] . " - ‚Ç±" . $product['price'] . "<br>"; } // Output: // Biogesic - ‚Ç±5.50 // Neozep - ‚Ç±8.00 // Bioflu - ‚Ç±12.00

üåü Special PHP Variables (Superglobals)

PHP has built-in variables that are available everywhere. These are called "superglobals".

üì• $_GET - URL Parameters

Data passed through the URL (visible in address bar)

// URL: product.php?id=123&category=medicine $id = $_GET['id']; // Gets: 123 $category = $_GET['category']; // Gets: medicine

üì§ $_POST - Form Data

Data sent from forms (hidden from URL, more secure)

// Form with method="POST" // <input name="username" value="Juan"> $username = $_POST['username']; // Gets: Juan $password = $_POST['password']; // Gets form password

üç™ $_SESSION - User Session Data

Data that persists across pages while user is logged in

// Store user info after login $_SESSION['user_id'] = 42; $_SESSION['username'] = "Maria"; $_SESSION['role'] = "admin"; // Access on any page if ($_SESSION['role'] === 'admin') { // Show admin features }

üìÅ $_FILES - Uploaded Files

Information about files uploaded through forms

// Form with: <input type="file" name="photo"> $fileName = $_FILES['photo']['name']; // Original name $fileSize = $_FILES['photo']['size']; // Size in bytes $tempPath = $_FILES['photo']['tmp_name']; // Temp location

üñ•Ô∏è $_SERVER - Server Information

Information about the server and current request

$_SERVER['REQUEST_METHOD']; // GET or POST $_SERVER['PHP_SELF']; // Current file name $_SERVER['HTTP_HOST']; // Domain name $_SERVER['REMOTE_ADDR']; // User's IP address

üîç Checking Variable Types

PHP provides functions to check what type a variable is:

$name = "Maria"; $age = 25; $price = 99.99; $isActive = true; $nothing = null; $items = [1, 2, 3]; // Check functions return true/false is_string($name); // true - it's a string is_int($age); // true - it's an integer is_float($price); // true - it's a float is_bool($isActive); // true - it's a boolean is_null($nothing); // true - it's null is_array($items); // true - it's an array // Check if variable exists and has value isset($name); // true - exists and not null empty($name); // false - has a value // Get the type as string gettype($name); // "string" gettype($age); // "integer"
üí° Pro Tip: Always Check Before Using

Before using $_GET or $_POST data, always check if it exists:

// Safe way to get URL parameter $id = $_GET['id'] ?? null; // Use null if not set // Or check with isset if (isset($_GET['id'])) { $id = $_GET['id']; }

üîÑ Converting Between Types

Sometimes you need to change a value from one type to another:

$string_number = "123"; // This is a string $real_number = (int)$string_number; // Now it's integer 123 // Cast to different types: (int) // Convert to integer (float) // Convert to decimal number (string) // Convert to text (bool) // Convert to true/false (array) // Convert to array // Examples: $price = (float)"99.99"; // String to float: 99.99 $text = (string)100; // Integer to string: "100" $flag = (bool)1; // Integer to boolean: true $flag2 = (bool)0; // Integer to boolean: false // Using functions: intval("42"); // Convert to integer floatval("3.14"); // Convert to float strval(100); // Convert to string
‚ö†Ô∏è Why This Matters

Data from forms ($_GET, $_POST) is ALWAYS a string, even if it looks like a number! If you're doing math, convert it first.

// ‚ùå Problem: String math can cause issues $quantity = $_POST['quantity']; // "5" (string) // ‚úÖ Solution: Convert first $quantity = (int)$_POST['quantity']; // 5 (integer)
