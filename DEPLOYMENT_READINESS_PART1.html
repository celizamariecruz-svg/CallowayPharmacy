Critical Security Issues

Must be fixed BEFORE deployment

‚õî
DEPLOYMENT BLOCKED

The following issues MUST be resolved before the system can safely go live. Deploying with these issues could lead to data breaches, system compromise, or financial loss.

1. Hardcoded Database Credentials

The database credentials are hardcoded directly in the config file with an empty password.

// File: config.php define('DB_HOST', 'localhost'); define('DB_USER', 'root'); // ‚ö†Ô∏è Default username define('DB_PASS', ''); // ‚ö†Ô∏è EMPTY PASSWORD! define('DB_NAME', 'calloway_pharmacy');

üîß Solution: Use environment variables with a .env file

// Install: composer require vlucas/phpdotenv // .env file (NEVER commit to Git) DB_HOST=localhost DB_USER=pharmacy_user DB_PASS=StrongP@ssw0rd!2025 DB_NAME=calloway_pharmacy // config.php - Updated require_once 'vendor/autoload.php'; $dotenv = Dotenv\Dotenv::createImmutable(__DIR__); $dotenv->load(); define('DB_HOST', $_ENV['DB_HOST']); define('DB_USER', $_ENV['DB_USER']); define('DB_PASS', $_ENV['DB_PASS']); define('DB_NAME', $_ENV['DB_NAME']);

2. Debug Mode Enabled

Debug mode exposes detailed error messages to users, revealing system internals to potential attackers.

// File: config.php - CURRENT (DANGEROUS) define('DEBUG_MODE', true); // ‚ö†Ô∏è Shows all errors!

üîß Solution:

// For PRODUCTION define('DEBUG_MODE', false); ini_set('display_errors', '0'); ini_set('log_errors', '1'); ini_set('error_log', '/var/log/pharmacy/error.log');

3. Default Admin Password

The system ships with a well-known default password that attackers will try first.

// File: create_admin.php $password = 'admin123'; // ‚ö†Ô∏è Everyone knows this!

üîß Solution:

  • Force password change on first login
  • Implement password complexity requirements
  • Generate random initial password and email to admin

4. No HTTPS Enforcement

The application does not force secure connections, allowing credentials to be intercepted.

üîß Solution: Add to .htaccess

# Force HTTPS RewriteEngine On RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301] # Set secure cookie flag php_flag session.cookie_secure On php_flag session.cookie_httponly On
