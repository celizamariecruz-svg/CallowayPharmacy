üêò What is PHP?

PHP stands for "PHP: Hypertext Preprocessor" (yes, it's a recursive name!). It's a programming language specifically designed to build websites.

üí° Real World Analogy

Think of a restaurant:

  • HTML = The menu and decorations (what customers see)
  • CSS = The paint, furniture, ambiance (how it looks)
  • PHP = The kitchen (where the real work happens - preparing food, checking inventory)
  • Database = The storage room (where ingredients/data are kept)

Your customers (website visitors) only see the menu and restaurant decor. They never see the kitchen, but without it, there's no food!

üìù How PHP Code Starts

All PHP code must be wrapped in special tags. This tells the server "Hey, process this as PHP code!"

<?php // Your PHP code goes here echo "Hello, World!"; ?>
üìñ Line by Line Explanation
  • <?php - Opens PHP mode. Everything after this is PHP code.
  • // - Comment marker. Computer ignores this, it's just notes for humans.
  • echo - Command to "print" or "output" something.
  • "Hello, World!" - The text to display (in quotes).
  • ; - Semicolon ends the statement (like a period ends a sentence).
  • ?> - Closes PHP mode (optional at end of file).
‚ö†Ô∏è Common Beginner Mistake

Forgetting the semicolon ; at the end of lines! This is the #1 error for beginners. Every PHP statement MUST end with a semicolon.

üî£ Essential PHP Symbols

Let's learn what each symbol means in PHP. You'll see these everywhere!

$

Dollar Sign - Variable Marker

All variable names start with $. It tells PHP "this is a container for data."

$name = "John"; $age = 25; $price = 99.99;
=

Equals Sign - Assignment

Assigns a value to a variable. It means "put this value into this container."

// "Store 'Maria' in $name" $name = "Maria";
;

Semicolon - Statement Ender

Marks the end of a statement. Like a period at the end of a sentence.

$a = 1; // End of line 1 $b = 2; // End of line 2
" "

Double Quotes - Text String

Wraps text/words. Variables inside will show their values.

$name = "Juan"; echo "Hello, $name"; // Output: Hello, Juan
' '

Single Quotes - Literal Text

Also wraps text, but variables are NOT processed (shown as-is).

$name = "Juan"; echo 'Hello, $name'; // Output: Hello, $name
( )

Parentheses - Function Call / Grouping

Used when calling functions or grouping conditions.

strlen("hello"); // Call function if ($age > 18) { } // Condition
{ }

Curly Braces - Code Block

Groups multiple statements together. Used with if, while, function, etc.

if ($loggedIn) { echo "Welcome!"; showDashboard(); }
[ ]

Square Brackets - Array Access

Access items in arrays/lists by their index or key.

$fruits[0]; // First item $user['name']; // Get by key

üî£ More Important Symbols

//

Double Slash - Single Line Comment

Computer ignores everything after // on that line. Notes for humans.

// This is a comment $price = 100; // Price in pesos
/* */

Block Comment

For multi-line comments. Everything between /* and */ is ignored.

/* This is a multi-line comment */
.

Dot - String Concatenation

Joins/combines strings together. Like gluing words.

$first = "Hello"; $second = "World"; echo $first . " " . $second; // Output: Hello World
.=

Dot Equals - Append to String

Adds to the end of an existing string.

$message = "Hello"; $message .= " World"; // $message is now "Hello World"
->

Arrow - Object Access

Access methods or properties of an object.

$conn->query($sql); $result->fetch_assoc(); $stmt->affected_rows;
=>

Double Arrow - Array Key-Value

Connects a key to its value in arrays.

$user = [ 'name' => 'Maria', 'age' => 25 ];
::

Double Colon - Static Access

Access static methods/properties of a class without creating an object.

DateTime::createFromFormat(); PDO::FETCH_ASSOC;
??

Null Coalescing - Default Value

Use left value if it exists, otherwise use right value (default).

$name = $_GET['name'] ?? 'Guest'; // If name not provided, use 'Guest'

‚öñÔ∏è Comparison Symbols (True/False Tests)

These symbols are used to compare values and return true or false.

Symbol Name What it Does Example
== Equals Checks if values are equal (ignores type) 5 == "5" ‚Üí true
=== Identical Checks if values AND types are equal 5 === "5" ‚Üí false
!= Not Equal Checks if values are different 5 != 3 ‚Üí true
!== Not Identical Checks if values OR types are different 5 !== "5" ‚Üí true
> Greater Than Left is bigger than right 10 > 5 ‚Üí true
< Less Than Left is smaller than right 3 < 8 ‚Üí true
>= Greater or Equal Left is bigger or same as right 5 >= 5 ‚Üí true
<= Less or Equal Left is smaller or same as right 4 <= 5 ‚Üí true
‚ö†Ô∏è = vs == vs === Confusion
  • = (one equals) = Assignment - "Store this value"
  • == (two equals) = Comparison - "Are these equal?"
  • === (three equals) = Strict Comparison - "Are these exactly equal?"

Tip: Always use === when possible to avoid unexpected bugs!
