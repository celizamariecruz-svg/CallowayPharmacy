Q3: How did you implement the database structure?
A3: The database implementation follows a normalized structure with these key tables:

1. products Table:
   - product_id (Primary Key, Auto Increment)
   - name (VARCHAR(255), Indexed)
   - category (VARCHAR(100), Indexed)
   - stock_quantity (INT)
   - expiry_date (DATE, Indexed)
   - location (VARCHAR(100))
   - unit_price (DECIMAL(10,2))
   - supplier_id (Foreign Key)
   - is_active (BOOLEAN)
   - created_at (TIMESTAMP)
   - updated_at (TIMESTAMP)

2. users Table:
   - user_id (Primary Key, Auto Increment)
   - username (VARCHAR(50), Unique)
   - password_hash (VARCHAR(255))
   - role (ENUM('admin', 'staff', 'manager'))
   - email (VARCHAR(255))
   - last_login (TIMESTAMP)
   - is_active (BOOLEAN)

3. transactions Table:
   - transaction_id (Primary Key, Auto Increment)
   - product_id (Foreign Key)
   - user_id (Foreign Key)
   - transaction_type (ENUM('in', 'out'))
   - quantity (INT)
   - transaction_date (TIMESTAMP)
   - notes (TEXT)

4. suppliers Table:
   - supplier_id (Primary Key, Auto Increment)
   - name (VARCHAR(255))
   - contact_person (VARCHAR(100))
   - phone (VARCHAR(20))
   - email (VARCHAR(255))
   - address (TEXT)

Key Database Features:
- Foreign key constraints for referential integrity
- Indexes on frequently searched columns
- Triggers for inventory updates
- Stored procedures for complex operations
- Regular backups and transaction logging

Q5: How does the expiry monitoring system work?
A5: The expiry monitoring system utilizes a sophisticated tracking mechanism:

Technical Implementation:
1. Date Calculation:
   ```php
   $today = new DateTime();
   $expiryDate = new DateTime($row['expiry_date']);
   $diff = $today->diff($expiryDate)->format("%r%a");
   ```

2. Status Categories:
   - Expired: diff < 0 days
   - Expiring Soon: 0 ≤ diff ≤ 30 days
   - Valid: diff > 30 days

3. Visual Indicators:
   - Red: Expired items
   - Orange: Expiring Soon
   - Green: Valid items
   - Dynamic countdown display

4. Database Queries:
   ```sql
   SELECT product_id, name, category, stock_quantity, 
          expiry_date, location 
   FROM products 
   WHERE (is_active = 1 OR is_active IS NULL) 
   AND expiry_date IS NOT NULL 
   ORDER BY expiry_date ASC
   ```

5. Real-time Updates:
   - JavaScript date calculations
   - Automatic page refresh
   - WebSocket notifications (planned)
