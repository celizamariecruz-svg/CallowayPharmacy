üìä What is a Database?

A database is where your application stores all its data permanently. Think of it as a super organized digital filing cabinet.

üí° Real World Analogy

Your pharmacy database is like your physical storage system:

  • Database = The entire filing cabinet
  • Tables = Different drawers (Products, Sales, Customers)
  • Columns = Labels on folders (Name, Price, Stock)
  • Rows = Individual records (each product)

Example: Products Table

id name brand price stock expiry_date
1 Paracetamol 500mg Biogesic 5.50 150 2026-06-30
2 Cold Tablet Neozep 8.00 75 2025-12-31
3 Flu Relief Bioflu 12.00 200 2026-03-15

üîå Connecting to the Database

Before you can use the database, you need to connect to it. This is done in your db_connection.php file.

<?php // Database connection settings $host = "localhost"; // Where the database is (usually localhost) $username = "root"; // Database username $password = ""; // Database password (empty for XAMPP default) $database = "pharmacy_db"; // Name of your database // Create connection $conn = new mysqli($host, $username, $password, $database); // Check if connection worked if ($conn->connect_error) { // Something went wrong! die("Connection failed: " . $conn->connect_error); } // Connection successful - $conn is now your connection object echo "Connected successfully!"; ?>
üí° What Each Part Means
  • new mysqli() - Creates a new MySQL connection object
  • $conn - The connection object (you'll use this for all queries)
  • connect_error - Contains error message if connection failed
  • die() - Stops the script and shows a message

‚ûï CREATE - Adding New Data (INSERT)

Use INSERT to add new records to a table.

// Method 1: Simple (NOT RECOMMENDED - security risk!) $sql = "INSERT INTO products (name, brand, price, stock) VALUES ('Paracetamol', 'Biogesic', 5.50, 100)"; $conn->query($sql);
‚ö†Ô∏è Security Warning: SQL Injection!

Never put user input directly into SQL! Hackers can inject malicious code. Always use prepared statements.

// ‚úÖ CORRECT WAY: Prepared Statement $name = $_POST['name']; // User input: "Biogesic" $brand = $_POST['brand']; // User input: "Unilab" $price = $_POST['price']; // User input: "5.50" $stock = $_POST['stock']; // User input: "100" // Step 1: Prepare the SQL with placeholders (?) $stmt = $conn->prepare("INSERT INTO products (name, brand, price, stock) VALUES (?, ?, ?, ?)"); // Step 2: Bind the actual values to placeholders // "ssdi" = string, string, double, integer (types of each value) $stmt->bind_param("ssdi", $name, $brand, $price, $stock); // Step 3: Execute the query if ($stmt->execute()) { echo "Product added! ID: " . $conn->insert_id; } else { echo "Error: " . $stmt->error; } // Step 4: Close the statement $stmt->close();
üí° bind_param Type Codes
  • s = string (text)
  • i = integer (whole number)
  • d = double (decimal number)
  • b = blob (binary data, like files)

üìñ READ - Getting Data (SELECT)

Use SELECT to retrieve data from the database.

Get All Records

// Get all products $sql = "SELECT * FROM products"; $result = $conn->query($sql); // Check if we got results if ($result->num_rows > 0) { // Loop through each row while ($row = $result->fetch_assoc()) { echo $row['name'] . " - ‚Ç±" . $row['price'] . "<br>"; } } else { echo "No products found"; } // Output: // Paracetamol - ‚Ç±5.50 // Neozep - ‚Ç±8.00 // Bioflu - ‚Ç±12.00
üí° Understanding the Code
  • SELECT * = Get all columns (* means "everything")
  • $result = Contains all matching rows
  • num_rows = Count of rows found
  • fetch_assoc() = Get one row as an array with column names

Get Specific Records (with WHERE)

// Get products with low stock (less than 50) $sql = "SELECT * FROM products WHERE stock < 50"; // Get products by brand $sql = "SELECT * FROM products WHERE brand = 'Biogesic'"; // Get products expiring soon $sql = "SELECT * FROM products WHERE expiry_date < '2025-06-01'"; // Multiple conditions $sql = "SELECT * FROM products WHERE stock < 50 AND price > 10";

Get Single Record (with Prepared Statement)

// Get product by ID (from URL: product.php?id=5) $id = $_GET['id']; $stmt = $conn->prepare("SELECT * FROM products WHERE id = ?"); $stmt->bind_param("i", $id); // "i" = integer $stmt->execute(); $result = $stmt->get_result(); $product = $result->fetch_assoc(); // Get single row if ($product) { echo "Product: " . $product['name']; echo "Price: ‚Ç±" . $product['price']; } else { echo "Product not found"; }

‚úèÔ∏è UPDATE - Modifying Data

Use UPDATE to change existing records.

// Update product price and stock $id = $_POST['id']; $newPrice = $_POST['price']; $newStock = $_POST['stock']; $stmt = $conn->prepare("UPDATE products SET price = ?, stock = ? WHERE id = ?"); $stmt->bind_param("dii", $newPrice, $newStock, $id); if ($stmt->execute()) { if ($stmt->affected_rows > 0) { echo "Product updated successfully!"; } else { echo "No changes made (maybe same values?)"; } } else { echo "Error: " . $stmt->error; }
‚ö†Ô∏è Always Use WHERE!

Without WHERE, you'll update ALL rows! Always specify which row(s) to update.

// ‚ùå DANGEROUS - Updates ALL products! "UPDATE products SET price = 0" // ‚úÖ SAFE - Only updates product #5 "UPDATE products SET price = 0 WHERE id = 5"

üóëÔ∏è DELETE - Removing Data

Use DELETE to remove records from the database.

// Delete a product by ID $id = $_POST['id']; $stmt = $conn->prepare("DELETE FROM products WHERE id = ?"); $stmt->bind_param("i", $id); if ($stmt->execute()) { if ($stmt->affected_rows > 0) { echo "Product deleted!"; } else { echo "Product not found"; } }
‚ö†Ô∏è Double Warning!

DELETE is permanent! Once data is deleted, it's gone forever. Same as UPDATE - always use WHERE!

üìö SQL Keywords Reference

Here are the most common SQL keywords you'll see in your code:

-- SELECT: Choose which columns SELECT * -- All columns SELECT name, price -- Specific columns SELECT COUNT(*) -- Count rows SELECT SUM(price) -- Add up values -- FROM: Which table FROM products FROM sales -- WHERE: Filter rows WHERE id = 5 -- Exact match WHERE price > 100 -- Greater than WHERE name LIKE '%bio%' -- Contains 'bio' WHERE stock BETWEEN 10 AND 50 -- ORDER BY: Sort results ORDER BY name ASC -- A to Z ORDER BY price DESC -- Highest first -- LIMIT: Restrict results LIMIT 10 -- Only first 10 LIMIT 10 OFFSET 20 -- Skip 20, get next 10 -- JOIN: Combine tables JOIN categories ON products.category_id = categories.id -- GROUP BY: Group similar rows GROUP BY category -- Totals per category

üéØ Full Working Example

Here's a complete example that loads products into an HTML table:

<?php // Include database connection require_once 'db_connection.php'; // Get all products, sorted by name $sql = "SELECT * FROM products ORDER BY name ASC"; $result = $conn->query($sql); ?> <!DOCTYPE html> <html> <body> <h1>Products List</h1> <table border="1"> <tr> <th>Name</th> <th>Brand</th> <th>Price</th> <th>Stock</th> </tr> <?php while ($row = $result->fetch_assoc()): ?> <tr> <td><?= htmlspecialchars($row['name']) ?></td> <td><?= htmlspecialchars($row['brand']) ?></td> <td>‚Ç±<?= number_format($row['price'], 2) ?></td> <td><?= $row['stock'] ?></td> </tr> <?php endwhile; ?> </table> <p>Total Products: <?= $result->num_rows ?></p> </body> </html> <?php // Close connection $conn->close(); ?>
üí° New Things in This Example
  • <?= ... ?> = Short for <?php echo ... ?>
  • htmlspecialchars() = Security function that prevents XSS attacks
  • number_format() = Formats number with commas and decimals
  • while(): / endwhile; = Alternative loop syntax for HTML templates
